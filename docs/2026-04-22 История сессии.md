### 3. Стабильность и автозапуск

**Задача пользователя:** сайт запущен стабильно? при перезагрузке системы он будет работать?

**Диагностика:**
- Docker-контейнеры: `restart: unless-stopped` ✅ — переживут перезагрузку
- Next.js dev-сервер: запущен вручную ❌ — не переживёт

**Решение — PM2:**
```bash
sudo npm install -g pm2
npm run build           # production сборка
pm2 start ecosystem.config.js
pm2 save
pm2 startup → sudo ... # регистрация в systemd
```

**Результат:** создан systemd-сервис `pm2-ai-openyoga.service`, все компоненты запускаются автоматически при перезагрузке.

**Результат первого прогона:** ✅ 30 / ⚠️ 7 / ❌ 7

**Найдены и исправлены баги:**

| # | Баг | Причина | Фикс |
|---|-----|---------|------|
| 1 | NextAuth CSRF 500 | `AUTH_SECRET` и `AUTH_TRUST_HOST=true` не заданы (NextAuth v5 требует их в production) | Добавлены в `.env.local` |
| 2 | CryptoCloud webhook 500 | `CRYPTOCLOUD_WEBHOOK_SECRET` отсутствовал в `.env.local` | Добавлен |
| 3 | `GET /api/admin/products` → 405 | Существовал только POST-хендлер | Создан GET-хендлер с пагинацией и фильтрацией |
| 4 | `GET /api/admin/analytics` → 500 | catch проверял `e.status`, а `requireRole` бросает `Error('UNAUTHORIZED')` без `.status` | Исправлен catch: проверка `e.message` |
| 5 | `GET /api/products` → 404 | Маршрут листинга каталога отсутствовал | Создан `/api/products/route.ts` |
| 6 | `GET /api/products/:slug` → 404 | Не было root-хендлера в `/api/products/[id]/` | Создан `/api/products/[id]/route.ts` |
| 7 | OpenGraph теги отсутствовали | Не добавлены в `app/layout.tsx` | Добавлены `openGraph` и `twitter` в metadata |
| 8 | `/catalog` → 500 | WAVE_PALETTE из client-модуля в Server Component | Вынесен в `lib/wave-palette.ts` (исправлен ранее) |

**Результат финального прогона:** ✅ 41 / ⚠️ 3 (ложные) / ❌ 0

Ложные срабатывания (не баги):
- `/api/cart` → 404 — корзина в `localStorage`, API не нужен
- `/privacy` → 404 — правильный URL `/privacy-policy` (ошибка в тест-скрипте)
- Сессия `null` через curl — корректно для JWT без cookies
