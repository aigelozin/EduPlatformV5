# EduPlatform РФ — Architecture Decision Records (ADR)

Адаптировано из V5.1. Документирует решения, специфичные для российской инфраструктуры.

---

## ADR-РФ-001: Яндекс Object Storage вместо Cloudflare R2

**Статус:** Принято

**Контекст:** Cloudflare — американская компания. Данные в R2 хранятся на серверах в США/ЕС.
ФЗ-152 требует хранить персональные данные россиян на серверах в РФ.
Видео-материалы и учебные файлы приравниваются к пользовательским данным.

**Решение:** Яндекс Object Storage (YOS) — S3-совместимый сервис, дата-центры в Москве.
SDK: `@aws-sdk/client-s3` с кастомным endpoint `https://storage.yandexcloud.net`.

**Последствия:**
- API идентичен S3/R2 — смена только endpoint и credentials.
- Бесплатного tier нет (в отличие от R2), но стоимость минимальна: ~2 руб/ГБ/мес.
- `generateSignedYOSUrl()` заменяет `generateSignedR2Url()` — логика идентична.
- Публичный бакет для аватаров/обложек, приватный для защищённого контента.

## ADR-РФ-002: Яндекс Managed Redis вместо Upstash

**Статус:** Принято

**Контекст:** Upstash — американская компания, данные хранятся за рубежом.
Beget Shared не предоставляет Redis. Нужен managed Redis в РФ.

**Решение:** Яндекс Managed Service for Redis (Yandex Cloud).
Минимальный тариф: ~750 руб/мес, дата-центры в Москве.

**Последствия:**
- Подключение через стандартный `ioredis` или `redis` npm пакет.
- Замена Upstash REST SDK на прямое Redis-подключение.
- Rate limiting middleware нужно переписать с Upstash REST на стандартный Redis client.
- В Docker Compose для локальной разработки Redis уже есть — изменений не нужно.

## ADR-РФ-003: SendPulse вместо Resend

**Статус:** Принято

**Контекст:** Resend — американский сервис. Для ФЗ-152 предпочтительны российские провайдеры.

**Решение:** SendPulse (российская компания, данные в РФ).
Бесплатно: 15 000 писем/мес. Есть REST API для транзакционных писем.
Beget SMTP — как резервный вариант (встроен в хостинг, бесплатно).

**Последствия:**
- Другой API по сравнению с Resend — нужен отдельный `lib/email/sendpulse.ts`.
- Beget SMTP как fallback через nodemailer.
- Шаблоны писем нужно адаптировать под SendPulse формат.

## ADR-РФ-004: GlitchTip вместо Sentry

**Статус:** Принято

**Контекст:** Sentry — американская компания, данные об ошибках (включая stack traces с данными пользователей) хранятся в США.

**Решение:** GlitchTip — open source альтернатива Sentry, self-hosted на собственном сервере в РФ.
Полностью совместим с Sentry SDK — замена DSN URL, никаких изменений в коде.

**Последствия:**
- Требуется отдельный сервер или Docker на Beget VPS для GlitchTip.
- На Beget Shared запустить Docker нельзя — нужен VPS (Timeweb/RuVDS/Beget VPS).
- Альтернатива: Яндекс Мониторинг (менее функциональный, но полностью в Яндекс Облаке).

## ADR-РФ-005: Убран PayPal

**Статус:** Принято

**Контекст:** PayPal ненадёжно работает с российскими аккаунтами с 2022 года.
Риск блокировки платежей, возвратов, заморозки средств.
Остаётся 3 провайдера для покрытия всех сегментов аудитории РФ.

**Решение:** PayPal полностью удалён.
Оставшиеся провайдеры:
- **YooKassa** — карты Visa/MC/МИР, СБП, ЮMoney, кошельки
- **МИР Pay** — карты МИР напрямую через НСПК
- **CryptoCloud** — USDT, BTC, ETH и др.

**Последствия:**
- `lib/payments/paypal.ts` удалён.
- `app/api/webhooks/paypal/route.ts` удалён.
- `PAYPAL_*` env vars удалены из `.env.example`.
- `payment_methods_config` seed не включает PayPal.

## ADR-РФ-006: Убрана международная доставка (EasyPost)

**Статус:** Принято

**Контекст:** Проект ориентирован на рынок РФ. Международная доставка из РФ в 2024–2026 значительно осложнена санкциями, ограничениями SWIFT и логистическими проблемами.

**Решение:** EasyPost удалён. Остаются CDEK и Boxberry, покрывающие РФ + СНГ.

**Последствия:**
- `lib/delivery/easypost.ts` удалён.
- `orders.delivery_service` enum: только `cdek | boxberry | manual`.
- CDEK покрывает часть стран СНГ — достаточно для целевой аудитории.

## ADR-РФ-007: YandexGPT вместо Claude API (primary)

**Статус:** Принято

**Контекст:** Claude API — Anthropic (US). Данные запросов могут обрабатываться вне РФ.
YandexGPT — Яндекс (РФ), данные обрабатываются в России.

**Решение:** YandexGPT API как primary AI провайдер.
Claude API остаётся как fallback — для SEO-генерации или если YandexGPT недоступен.

**Последствия:**
- `lib/ai/yandexgpt.ts` — основной клиент.
- `lib/ai/claude.ts` — fallback, используется только при недоступности YandexGPT.
- API YandexGPT отличается от Anthropic API — нужен отдельный клиент.
- YandexGPT Lite быстрее и дешевле, YandexGPT лучше по качеству.
- Для чата: YandexGPT. Для SEO: YandexGPT Pro или Claude API.

## ADR-РФ-008: VK Video и RuTube как основные источники видео

**Статус:** Принято

**Контекст:** YouTube в РФ с 2024 значительно замедлен (~40–80 kbps на мобильных).
Archive.org недоступен из РФ. Нужны надёжные российские видеохостинги.

**Решение:**
- **VK Video** — основной бесплатный вариант, стабильный в РФ
- **RuTube** — второй основной вариант, государственная платформа
- **Kinescope** — платный (~$30/мес), но лучшее качество, аналитика, защита от скачивания
- **YouTube** — запасной источник (может быть замедлен)
- **YOS** — для собственных загруженных видео

**Последствия:**
- `video_source` enum: `vk | rutube | kinescope | youtube | yos`
- `archive` значение удалено.
- VideoPlayer компонент адаптирован под 5 источников (без Archive.org).

## ADR-РФ-009: Только русский язык (убран EN)

**Статус:** Принято

**Контекст:** Аудитория — РФ и русскоязычные пользователи. Поддержка EN с первого дня удваивает объём работы и усложняет контент-менеджмент для преподавателей.

**Решение:** next-intl остаётся, но только с `ru` локалью.
Поля `*_en` в БД — опциональны (не удалены для возможного будущего расширения), по умолчанию `null`.

**Последствия:**
- `messages/en.json` — не создаётся.
- URL-структура: без `/en/` префикса.
- SEO только на русском языке.
- Упрощение форм преподавателя — не нужно заполнять EN поля.
