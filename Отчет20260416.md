## 1. ОПИСАНИЕ ПРОЕКТА

Образовательный маркетплейс с несколькими преподавателями (йога, массаж, фитнес, творчество, бизнес).  
Поддерживает: цифровой контент, физические товары, подписки, прямые трансляции.  
Юрисдикция: ООО в России (ФЗ-152). Вся инфраструктура — российские сервисы.

**Стек:**
- Next.js 14 App Router + TypeScript strict
- PostgreSQL (Beget Shared) + Prisma ORM (23 таблицы)
- NextAuth.js v5 (JWT сессии)
- Yandex Object Storage (файловое хранилище, ФЗ-152)
- Платежи: YooKassa + CryptoCloud + МИР Pay
- Доставка: CDEK API v2 + Boxberry
- AI: YandexGPT (основной) + Claude API (запасной)
- Redis: Яндекс Managed Redis (rate limiting)
- Email: SendPulse + Beget SMTP (fallback)
- Мониторинг: GlitchTip self-hosted
- Деплой: Beget Shared + PM2 + Nginx + GitHub Actions CI/CD

          БЛОК D (тесты) → деплой с полным функционалом

## 9. ВАЖНЫЕ ПАТТЕРНЫ / ПРАВИЛА РАЗРАБОТКИ

1. **try/catch на всех DB вызовах** в Server Components — страницы показывают пустое состояние вместо 500
2. **Middleware trailing slash**: `/teacher/` а не `/teacher` — иначе ломается `/teachers`
3. **Без intl middleware** — использовать только `next-intl/server` (getMessages, getLocale) напрямую
4. **DEV_BYPASS_AUTH** — УБРАТЬ перед деплоем на Beget
5. **checkAccess()** — обязателен перед любым контентом/видео
6. **Изоляция преподавателя** — `WHERE creator_id = session.user.id` в каждом teacher запросе
7. **Подпись webhook** — верифицировать до обработки payload
8. **ФЗ-152** — consent_log при регистрации, checkout, регистрации преподавателя
